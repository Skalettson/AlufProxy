# Политика конфиденциальности AlufProxy

**Последнее обновление:** 14 марта 2026 г.

---

## 1. Введение

Настоящая Политика конфиденциальности описывает, как **AlufProxy** («мы», «нас», «наш») собирает, использует, хранит и защищает вашу информацию при использовании нашего программного обеспечения.

**Мы серьёзно относимся к вашей приватности** и стремимся минимизировать сбор персональных данных.

---

## 2. Сбор информации

### 2.1. Информация, которую мы НЕ собираем

**AlufProxy принципиально НЕ собирает:**

- ❌ Реальные имена и фамилии
- ❌ Адреса электронной почты
- ❌ Номера телефонов
- ❌ Физические адреса
- ❌ Платёжную информацию
- ❌ Историю посещённых сайтов
- ❌ Содержимое вашего трафика
- ❌ Геолокационные данные
- ❌ Данные с устройства (камера, микрофон, файлы)

### 2.2. Информация, которую мы собираем

#### Telegram-бот

При использовании Telegram-бота (@AlufProxyBot) автоматически собирается:

| Данные | Цель | Хранение |
|--------|------|----------|
| **Telegram ID** | Идентификация пользователя | SQLite БД на сервере бота |
| **Username (@name)** | Отображение в интерфейсе | SQLite БД, до удаления |
| **Имя (first_name)** | Персонализация ответов | SQLite БД, до удаления |
| **Дата регистрации** | Учёт пробного периода | SQLite БД, бессрочно |
| **Дата окончания подписки** | Контроль доступа | SQLite БД, до истечения |
| **Сгенерированные ключи** | Предоставление доступа | SQLite БД, до отзыва |

#### Клиентское приложение (Windows)

Клиентское приложение **не передаёт данные разработчикам**. Локально хранятся:

| Данные | Цель | Хранение |
|--------|------|----------|
| **VLESS ключи** | Подключение к серверу | `%APPDATA%\AlufProxy\config.json` |
| **Настройки порта** | Конфигурация прокси | `%APPDATA%\AlufProxy\config.json` |
| **Настройки автозапуска** | Удобство использования | `%APPDATA%\AlufProxy\config.json` |
| **Логи работы** | Отладка (опционально) | `%APPDATA%\AlufProxy\alufproxy.log` |

#### Серверная часть (VPS)

Администратор сервера может настроить логирование:

| Данные | Цель | Примечание |
|--------|------|------------|
| **Время подключений** | Мониторинг нагрузки | Опционально |
| **Объём трафика** | Учёт использования | Опционально |
| **IP-адреса** | Безопасность (anti-DDoS) | Опционально, не рекомендуется |

**Важно:** Настройка логирования осуществляется **владельцем сервера**, а не разработчиками.

## 3. Цели использования информации

Мы используем собранную информацию исключительно для:

### 3.1. Функционирование сервиса

- Идентификация пользователей в Telegram-боте
- Генерация и учёт ключей доступа
- Контроль срока действия подписок
- Предоставление технической поддержки

### 3.2. Безопасность

- Предотвращение злоупотреблений
- Защита от DDoS-атак (на уровне сервера)
- Блокировка нарушителей Условий использования

### 3.3. Улучшение сервиса

- Анализ статистики использования (агрегированные данные)
- Выявление и исправление ошибок
- Разработка новых функций

## 4. Хранение данных

### 4.1. Место хранения

| Компонент | Расположение | Доступ |
|-----------|--------------|--------|
| **Telegram-бот (БД)** | Сервер Vercel (AWS/Cloudflare) | Разработчики бота |
| **Клиентское приложение** | Локально на устройстве пользователя | Только пользователь |
| **Сервер XRay** | Выбранный пользователем VPS | Владелец VPS |

### 4.2. Срок хранения

| Данные | Срок хранения |
|--------|---------------|
| Telegram ID, username | До удаления пользователем (/stop) |
| Ключи доступа | До истечения срока действия + 30 дней |
| Логи клиента | До очистки пользователем |
| Логи сервера | По настройкам владельца сервера |

### 4.3. Защита данных

Мы применяем следующие меры защиты:

- **Шифрование** — ключи хранятся в зашифрованном виде
- **Доступ по необходимости** — ограниченный доступ к БД
- **Регулярные бэкапы** — резервное копирование БД
- **Минимизация данных** — храним только необходимое

## 5. Передача данных третьим лицам

### 5.1. Когда мы передаём данные

**Мы НЕ продаём и не передаём данные третьим лицам**, за исключением:

#### Обязательные случаи (по закону):
- По официальному запросу правоохранительных органов
- По решению суда
- В случаях, предусмотренных законодательством РФ

#### Технические необходимости:
- **Vercel** — хостинг Telegram-бота (данные обрабатываются на их серверах)
- **Telegram** — платформа для бота (данные обрабатываются по их политике)
- **Хостинг-провайдер VPS** — только если вы используете их сервер

### 5.2. Когда мы НЕ передаём данные

Мы **никогда не передаём** данные:
- Рекламным сетям
- Аналитическим сервисам
- Маркетинговым компаниям
- Страховым компаниям
- Работодателям

## 6. Ваши права

Как пользователь, вы имеете право:

### 6.1. Право на доступ

Вы можете запросить копию ваших данных:
- Напишите в поддержку @aluf_support
- Укажите ваш Telegram ID
- Получите данные в течение 7 дней

### 6.2. Право на исправление

Вы можете исправить неточные данные:
- Через команды бота
- Через обращение в поддержку

### 6.3. Право на удаление

Вы можете удалить ваши данные:

**Через бота:**
```
/stop — остановить бота
/delete — удалить аккаунт и данные
```

**Через поддержку:**
- Напишите @aluf_support с просьбой об удалении
- Данные будут удалены в течение 14 дней

### 6.4. Право на перенос

Вы можете экспортировать ваши данные:
- Ключи доступа сохраняются в боте
- Локальные настройки — в `%APPDATA%\AlufProxy\`

### 6.5. Право на возражение

Вы можете возразить против обработки:
- Отзовите согласие через /stop
- Удалите приложение
- Обратитесь в поддержку

## 7. Cookies и аналогичные технологии

### 7.1. Cookies

AlufProxy **не использует** cookies.

### 7.2. Аналогичные технологии

Мы **не используем**:
- Пиксели отслеживания
- Fingerprints браузера
- LocalStorage для отслеживания
- Аналитические скрипты

---

## 8. Безопасность

### 8.1. Наши меры

Мы принимаем меры для защиты данных:

- **Шифрование при передаче** — TLS 1.3 для всех соединений
- **Шифрование при хранении** — ключи шифруются в БД
- **Регулярные обновления** — своевременное обновление зависимостей
- **Минимизация доступа** — доступ только для необходимых лиц

### 8.2. Ваши меры безопасности

Рекомендуем вам:

- **Не делитесь ключами** — ключи персональные
- **Используйте надёжные пароли** — для Telegram-аккаунта
- **Включите 2FA** — двухфакторная аутентификация в Telegram
- **Регулярно обновляйте** — клиентское приложение
- **Проверяйте подлинность** — скачивайте только из официальных источников

### 8.3. Уведомление об утечках

В случае утечки данных мы:
- Уведомим пользователей в течение 72 часов
- Опубликуем информацию в репозитории
- Предоставим рекомендации по защите

### 9.1. Возрастные ограничения

AlufProxy **не предназначен** для использования лицами младше 16 лет.

### 9.2. Сбор данных детей

Мы **не собираем сознательно** данные детей:

## 10. Международная передача данных

### 10.1. География обработки

Данные могут обрабатываться в:
- **Российская Федерация** — разработчики
- **США/ЕС** — серверы Vercel (AWS, Cloudflare)
- **Страна VPS** — выбранный пользователем сервер

### 10.2. Защита при передаче

При международной передаче:
- Данные шифруются (TLS)
- Применяются стандартные договорные положения
- Соблюдается GDPR (для пользователей из ЕС)
